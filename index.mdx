## What is Solver Rewards?

CoW Protocol Solver Rewards is a Python-based data aggregation and accounting tool that automates the weekly computation and distribution of solver reimbursements and COW token rewards across multiple EVM-compatible networks.

Solvers — the entities that find optimal trade routes for CoW Protocol — receive two forms of compensation each week:

- **Native token reimbursements** — gas and execution cost coverage paid in ETH (or the chain's native token)
- **COW token rewards** — performance-based rewards paid in COW tokens for batch settlement and quote provision

The system aggregates data from [Dune Analytics](https://dune.com) and the CoW Protocol orderbook analytics database, computes per-solver payouts, and either generates CSV transfer files for manual execution or posts multisend transactions directly to a Gnosis Safe.

## Key features

<AccordionGroup>
  <Accordion title="Weekly automated payout calculation">
    The accounting period runs in 7-day intervals, typically executed each Tuesday. The script aggregates slippage, batch rewards, quote rewards, protocol fees, and network fees per solver over the period.
  </Accordion>
  <Accordion title="Dual-token reward model">
    Solvers receive native token reimbursements (ETH, xDAI, etc.) for execution costs and COW token rewards for performance. The split and exchange rates are computed automatically using Coinpaprika price data.
  </Accordion>
  <Accordion title="Overdraft tracking">
    When a solver's slippage exceeds their rewards, the deficit is recorded as an overdraft in an on-chain contract rather than resulting in a negative transfer.
  </Accordion>
  <Accordion title="Gnosis Safe integration">
    Payouts can be posted directly as multisend transactions to the CoW Team Safe, or exported as CSV files compatible with the Safe Airdrop app.
  </Accordion>
  <Accordion title="Service fee deduction">
    A 15% service fee is deducted from solver rewards before distribution, directed to CoW DAO. This is applied to both primary performance rewards and quote rewards.
  </Accordion>
  <Accordion title="Partner fee distribution">
    Protocol fees collected from partner integrations are distributed to partner addresses after tax deductions, with the remainder going to the protocol fee safe.
  </Accordion>
</AccordionGroup>

## How it works

<Steps>
  <Step title="Data aggregation">
    The system queries Dune Analytics for on-chain block ranges, slippage data, and solver metrics. Simultaneously, it queries the CoW Protocol analytics database for batch reward, quote reward, and fee data.
  </Step>
  <Step title="Payout computation">
    Per-solver payouts are computed by combining slippage, batch rewards, quote rewards, protocol fees, and network fees. Exchange rates are fetched from Coinpaprika to convert between token denominations.
  </Step>
  <Step title="Overdraft detection">
    Solvers with a net negative position (where slippage exceeds total rewards) are flagged as overdrafts and recorded to an on-chain contract instead of receiving a transfer.
  </Step>
  <Step title="Transfer generation">
    Valid payouts are split into native token transfers (reimbursements) and COW token transfers (rewards), filtered by minimum transfer thresholds.
  </Step>
  <Step title="Execution">
    Transfers are either written to CSV files for manual Safe execution, or posted directly as multisend transactions to the Gnosis Safe using the proposer account.
  </Step>
</Steps>
